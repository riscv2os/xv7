# 修復報告 3：Shell 歷史命令顯示問題

## 問題描述

按上鍵瀏覽歷史命令時，歷史命令沒有正確顯示在當前行，而是會出現在下一行，導致螢幕上會看到重複的提示符。

## 原因分析

原始代碼在處理上/下箭頭鍵時，使用 `\b`（退格）來嘗試清除當前輸入，但這個方法無法正確處理以下情況：

1. 沒有先將光標移回行首
2. 沒有清除 `$ ` 提示符
3. 輸出邏輯混亂，導致提示符被重複輸出

## 修復方案

修改 `user/sh.c` 中的 `getcmd_with_history` 函數，改用更可靠的方式處理歷史命令顯示：

1. 使用 `\b \b`（退格 + 空格 + 退格）來清除當前輸入的每個字元
2. 清除完成後，直接輸出歷史命令

### 修改後的代碼

```c
if(c == 'A' || c == 'B'){
  if(c == 'A'){
    if(hist_index > 0) hist_index--;
  } else {
    if(hist_index < hist_count) hist_index++;
  }

  while(i > 0){
    write(2, "\b \b", 3);
    i--;
  }
  memset(buf, 0, nbuf);
  if(hist_index < hist_count){
    char *h = history[hist_index % HIST_SIZE];
    int hlen = strlen(h);
    if(hlen > nbuf - 1) hlen = nbuf - 1;
    memmove(buf, h, hlen);
    i = hlen;
    write(2, buf, i);
  }
  escape_state = 0;
  escape_len = 0;
  goto get_next_char;
}
```

## 修復結果

- 按上鍵可正確顯示上一個命令
- 按下鍵可正確顯示下一個命令
- 歷史命令正確替換當前輸入

## 已知限制

按上鍵時，終端會輕微向上滾動一行，這是終端的正常行為，不影響功能使用。
