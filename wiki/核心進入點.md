# 核心進入點

核心進入點負責作業系統的初始化，從硬體跳躍到核心空間開始，逐步初始化各個子系統。

## 開機流程

### 1. 開機 ROM (0x1000)

QEMU 載入後會先執行開機 ROM，位於 0x1000 位址。開機 ROM 負責：
- 載入核心映像檔至 0x80000000
- 設定必要的硬體
- 跳躍至核心入口

### 2. 核心入口 (kernel/entry.S)

```asm
# entry.S
.globl entry
entry:
    # 設定堆疊指標
    la sp, stack0
    # 跳躍到 main()
    call main
```

### 3. 主函數 (kernel/main.c)

核心主函數執行各項初始化：

```c
void main() {
    if(cpuid() == 0){
        consoleinit();      // 終端機初始化
        printfinit();       // 輸出初始化
        
        kinit();            // 實體記憶體配置器
        kvminit();          // 建立核心頁表
        kvminithart();      // 啟用分頁
        
        procinit();         // 行程表
        trapinit();         // 陷阱向量
        trapinithart();     // 安裝陷阱處理
        plicinit();         // 中斷控制器
        plicinithart();     // 設定中斷
        
        binit();            // 緩衝區快取
        iinit();            // inode 表
        fileinit();         // 檔案表
        virtio_disk_init(); // 磁碟驅動
        
        netinit();          // 網路堆疊 (xv7)
        virtio_net_init();  // 網路卡 (xv7)
        netrun();           // 啟動網路 (xv7)
        
        userinit();         // 第一個使用者行程
        started = 1;
    } else {
        // 其他 CPU 的初始化
        kvminithart();
        trapinithart();
        plicinithart();
    }
    
    scheduler();  // 啟動排程器
}
```

## 初始化順序

| 順序 | 元件 | 檔案 |
|------|------|------|
| 1 | 終端機 | kernel/console.c |
| 2 | 輸出 | kernel/printf.c |
| 3 | 記憶體配置 | kernel/kalloc.c |
| 4 | 虛擬記憶體 | kernel/vm.c |
| 5 | 行程管理 | kernel/proc.c |
| 6 | 陷阱處理 | kernel/trap.c |
| 7 | 中斷控制器 | kernel/plic.c |
| 8 | 緩衝區快取 | kernel/bio.c |
| 9 | 檔案系統 | kernel/fs.c |
| 10 | 磁碟驅動 | kernel/virtio_disk.c |
| 11 | 網路堆疊 | kernel/net/net.c |
| 12 | 網路卡 | kernel/net/platform/xv6-riscv/virtio_net.c |
| 13 | 使用者行程 | kernel/proc.c |

## 每個 CPU 的初始化

除了第一個 CPU (cpuid() == 0)，其他 CPU 執行簡化的初始化：

1. 等待 started 變數被設為 1
2. 初始化分頁 (kvminithart)
3. 設定陷阱向量 (trapinithart)
4. 設定中斷控制器 (plicinithart)

## 排程器啟動

最後呼叫 scheduler() 進入無窮迴圈，開始行程排程：

```c
void scheduler(void) {
    for(;;) {
        // 找到可執行的行程
        // 切換上下文執行
    }
}
```
