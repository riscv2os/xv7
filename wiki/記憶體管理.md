# 記憶體管理

xv6/xv7 的記憶體管理分為兩部分：虛擬記憶體管理（核心頁表、使用者頁表）和實體記憶體配置。

## 虛擬記憶體

### 核心頁表

核心使用單一的全域頁表 (`kernel_pagetable`)，採用直接映射：

```c
pagetable_t kernel_pagetable;
```

核心頁表映射：
- **直接映射**：虛擬位址 0x80000000 對應實體位址 0x80000000
- **UART**：0x10000000 → 0x10000000
- **VirtIO**：0x10001000 → 0x10001000
- **PLIC**：0x0C000000 → 0x0C000000
- **Trampoline**：最高虛擬位址

### 使用者頁表

每個行程有自己的使用者頁表：

```c
pagetable_t pagetable;  // 在 struct proc 中
```

使用者位址空間配置：
```
位址 0      → 程式碼 (text)
           → 資料 (data)
           → 堆疊 (stack)
           → 堆 (heap)
TRAPFRAME  → 陷阱框架頁面
TRAMPOLINE → 彈跳頁面
```

## 頁表結構

RISC-V SV39 使用三層頁表：
- **VPN[2]**：頂層索引
- **VPN[1]**：中層索引
- **VPN[0]**：底層索引
- **PPN**：實體頁號
- **Flags**：PTE_R, PTE_W, PTE_X, PTE_U

## 關鍵函數

### 核心虛擬記憶體
- `kvminit()` - 建立核心頁表
- `kvminithart()` - 啟用分頁
- `kvmmap(pagetable, va, pa, sz, perm)` - 映射虛擬到實體
- `walk(pagetable, va, alloc)` - 查詢頁表項目

### 使用者虛擬記憶體
- `proc_pagetable(struct proc *)` - 建立使用者頁表
- `uvmcreate()` - 建立空頁表
- `uvmalloc(pagetable, oldsz, newsz, perm)` - 配置新頁面
- `uvmdealloc(pagetable, oldsz, newsz)` - 釋放頁面
- `uvmcopy(pagetable, new, oldsz)` - 複製頁表
- `uvmfree(pagetable, sz)` - 釋放整個位址空間

## 實體記憶體配置

```c
void kinit(void);      // 初始化配置器
void *kalloc(void);    // 配置一個頁面
void kfree(void *);    // 釋放一個頁面
```

使用 free list 管理閒置頁面。

## 記憶體配置

| 位址範圍 | 用途 |
|---------|------|
| 0x1000 | 開機 ROM |
| 0x02000000 | CLINT |
| 0x0C000000 | PLIC |
| 0x10000000 | UART0 |
| 0x10001000 | VirtIO 磁碟 |
| 0x10002000 | VirtIO 網路 |
| 0x80000000 | 核心程式碼 |
| 0x80000000 + end | 核心動態配置 |
| PHYSTOP | 實體記憶體頂端 |
