# 行程管理

xv6/xv7 使用簡單的輪詢式排程器。每個行程都有自己的核心堆疊和使用者位址空間。

## 行程結構

```c
struct proc {
  enum procstate state;    // UNUSED, USED, SLEEPING, RUNNABLE, RUNNING, ZOMBIE
  void *chan;              // 睡眠通道
  int killed;              // 終止標記
  int xstate;              // 結束狀態
  int pid;                 // 行程 ID
  struct proc *parent;     // 父行程
  uint64 kstack;           // 核心堆疊位址
  uint64 sz;               // 使用者記憶體大小
  pagetable_t pagetable;   // 使用者頁表
  struct trapframe *trapframe;
  struct context context;  // 排程器上下文
  struct file *ofile[NOFILE];  // 開啟的檔案
  struct inode *cwd;       // 目前目錄
  char name[16];           // 行程名稱
};
```

## 行程狀態

| 狀態 | 說明 |
|------|------|
| `UNUSED` | 未配置 |
| `USED` | 已配置但不可執行 |
| `SLEEPING` | 等待 I/O 或事件 |
| `RUNNABLE` | 可執行 |
| `RUNNING` | 正在執行 |
| `ZOMBIE` | 已退出，等待父行程收集 |

## 關鍵函數

- `kfork()` - 建立新行程（複製目前）
- `kexit(int status)` - 終止目前行程
- `kwait(uint64 addr)` - 等待子行程退出
- `sleep(void *chan, struct spinlock *)` - 在通道上睡眠
- `wakeup(void *chan)` - 喚醒所有睡眠的行程
- `yield()` - 讓出 CPU 給其他行程

## 排程器

排程器持續執行以下循環：
1. 取得行程表鎖定
2. 找到 RUNNABLE 的行程
3. 呼叫 `swtch()` 切換到其上下文
4. 重複

## 上下文切換

```asm
# swtch.S - 組語常式
.global swtch
swtch:
  # 儲存目前上下文 (a0=舊, a1=新)
  sd ra, 0(a0)
  sd sp, 4(a0)
  # ... 儲存 callee-saved 暫存器
  
  # 恢復新上下文
  ld ra, 0(a1)
  ld sp, 4(a1)
  # ... 恢復 callee-saved 暫存器
  
  ret
```

## 行程相關系統呼叫

- `fork()` - 建立新行程
- `exit(int status)` - 結束目前行程
- `wait(uint64 addr)` - 等待子行程
- `kill(int pid)` - 終止行程
- `getpid()` - 取得目前行程 ID
- `sleep(uint ticks)` - 睡眠指定滴答數
