# 檔案系統

xv6/xv7 使用 SFS (Simple File System)，這是一個簡單的 Unix 風格磁碟檔案系統。

## 磁碟配置

```
+------------------+
|   啟動區塊       |  (boot block)
+------------------+
|   超級區塊      |  (super block)
+------------------+
|   日誌區塊      |  (log blocks)
+------------------+
|   inode 區塊    |  (inode blocks)
+------------------+
|   空閒位元圖    |  (free bitmap)
+------------------+
|   資料區塊      |  (data blocks)
+------------------+
```

## 超級區塊 (Superblock)

```c
struct superblock {
  uint magic;        // 必須是 FSMAGIC
  uint size;         // 檔案系統影像大小（區塊）
  uint nblocks;      // 資料區塊數量
  uint ninodes;      // inode 數量
  uint nlog;         // 日誌區塊數量
  uint logstart;     // 第一個日誌區塊編號
  uint inodestart;   // 第一個 inode 區塊編號
  uint bmapstart;    // 第一個空閒位元圖區塊編號
};
```

## inode 結構

```c
struct dinode {
  short type;           // 檔案類型
  short major;          // 主裝置編號 (T_DEVICE only)
  short minor;          // 次裝置編號 (T_DEVICE only)
  short nlink;          // 連結數
  uint size;            // 檔案大小（位元組）
  uint addrs[NDIRECT+1];   // 資料區塊位址
};
```

- **NDIRECT = 12**：直接區塊數量
- **NINDIRECT**：間接區塊數量（單一間接區塊）
- **MAXFILE**：最大檔案大小

## 目錄結構

```c
struct dirent {
  ushort inum;
  char name[DIRSIZ];  // DIRSIZ = 14
};
```

目錄是一個包含 dirent 結構的檔案。

## 緩衝區快取

磁碟 I/O 使用緩衝區快取 (`kernel/bio.c`)：

```c
struct buf {
  uint dev;           // 裝置編號
  uint blockno;       // 區塊編號
  uint flags;        // B_VALID, B_DIRTY
  struct sleeplock lock;
  uint refcnt;
  struct buf *prev;
  struct buf *next;
  struct buf *qnext;
  uchar data[BSIZE];  // 1024 位元組
};
```

## 關鍵函數

### 磁碟操作
- `bread(uint dev, uint blockno)` - 讀取區塊
- `bwrite(struct buf *)` - 寫入區塊
- `brelse(struct buf *)` - 釋放緩衝區

### inode 操作
- `ialloc(uint dev, short type)` - 配置新的 inode
- `idup(struct inode *)` - 複製 inode（增加連結數）
- `ilock(struct inode *)` - 鎖定 inode
- `iunlock(struct inode *)` - 解鎖 inode
- `iput(struct inode *)` - 釋放 inode
- `iupdate(struct inode *)` - 寫回磁碟

### 目錄操作
- `dirlookup(struct inode *, char *, uint *)` - 查詢目錄項目
- `dirlink(struct inode *, char *, uint)` - 新增目錄項目

## 日誌系統

xv6 使用簡單的日誌機制來確保檔案系統一致性：

1. **begin_op()** - 開始交易
2. **log_write(struct buf *)** - 寫入日誌
3. **end_op()** - 提交交易（寫入磁碟）
