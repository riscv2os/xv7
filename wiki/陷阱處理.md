# 陷阱處理

陷阱處理機制是作業系統的核心功能之一，負責處理系統呼叫、中斷和例外。

## 陷阱類型

RISC-V 將所有例外和中斷統一稱為「陷阱」：

| 類型 | 說明 | 範例 |
|------|------|------|
| 系統呼叫 | 使用者要求核心服務 | fork, exec, read, write |
| 例外 | 指令執行錯誤 | 非法記憶體存取、非法指令 |
| 中斷 | 外部事件觸發 | 計時器、UART、VirtIO |

## 陷阱向量

RISC-V 使用 `stvec` 暫存器儲存陷阱向量位址。xv6 使用以下配置：

```c
// kernel/trap.c
void trapinithart(void) {
    w_stvec((uint64)kernelvec);
}
```

### 使用者陷阱向量

使用者在 uservec (trampoline.S) 進入陷阱，儲存使用者暫存器後跳躍至 usertrap()。

### 核心陷阱向量

核心使用 kernelvec (kernelvec.S) 處理核心內的陷阱。

## 陷阱框架 (Trapframe)

每個行程有一個陷阱框架，儲存陷阱發生時的暫存器狀態：

```c
struct trapframe {
  /*   0 */ uint64 kernel_satp;   // 核心頁表
  /*   8 */ uint64 kernel_sp;     // 行程的核心堆疊
  /*  16 */ uint64 kernel_trap;   // 陷阱處理函數
  /*  24 */ uint64 epc;           // 儲存的使用者程式計數器
  /*  32 */ uint64 kernel_hartid; // 核心執行緒 ID
  /*  40 */ uint64 ra;
  /*  48 */ uint64 sp;
  /*  56 */ uint64 gp;
  /*  64 */ uint64 tp;
  /*  72 */ uint64 t0;
  // ... 其他暫存器
};
```

## 陷阱處理流程

### 使用者陷阱

1. **進入陷阱**：硬體自動切換到陷阱向量
2. **儲存狀態**：uservec 儲存使用者暫存器到 trapframe
3. **呼叫處理**：跳躍至 usertrap()
4. **處理原因**：判斷陷阱類型並處理
5. **系統呼叫**：呼叫對應的系統呼叫處理函數
6. **返回使用者**：usertrapret() 設定返回狀態
7. **恢復執行**：userret 恢復使用者暫存器，回到使用者空間

### 核心陷阱

1. **進入陷阱**：kernelvec 儲存暫存器
2. **處理異常**：根據 scause 判斷原因
3. **處理完成**：恢復執行或 panic

## 關鍵函數

```c
void trapinit(void);           // 初始化陷阱
void trapinithart(void);      // 設定陷阱向量
void usertrap(void);          // 使用者陷阱處理
void kerneltrap(void);        // 核心陷阱處理
void usertrapret(void);       // 返回使用者
```

## RISC-V 陷阱暫存器

| 暫存器 | 說明 |
|--------|------|
| `scause` | 陷阱原因 |
| `sepc` | 陷阱發生時的程式計數器 |
| `stvec` | 陷阱向量位址 |
| `sscratch` | 臨時儲存 |
| `sstatus` | 狀態（包含 SIE 開關） |

## 系統呼叫分派

```c
// kernel/syscall.c
void syscall(void) {
    int num = p->trapframe->a7;  // 系統呼叫編號
    if (num > 0 && num < NELEM(syscalls)) {
        p->trapframe->a0 = syscalls[num]();
    } else {
        p->trapframe->a0 = -1;
    }
}
```
